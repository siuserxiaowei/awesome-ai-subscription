# 贡献指南

谢谢你愿意帮忙改进这份指南。规则尽量简单。

## 可以提的 PR 类型

1. **价格更新** —— 各家每年至少调一次价,看到不一致直接改
2. **错别字 / 链接失效** —— 直接改,不需要先开 issue
3. **新付款渠道实测** —— 自己实际用过的,带成功率数据,放进 `06-payment-methods.md`
4. **新场景 / 新案例** —— 故障排查 (`08-troubleshooting.md`) 永远缺案例,欢迎补
5. **新订阅档位** —— 比如某家发了新 Tier,先开 issue 讨论命名和位置,再写

## 风格要求

- **中文为主**,技术名词保留英文(Plus / API / RAG / SOTA)
- **拒绝 AI 套话**:不要"在当今飞速发展的 AI 时代""综上所述""不仅...还..."
- **拒绝排比和总结句**:写"实测 50 单里 7 单失败"不写"经过大量测试发现"
- **多用具体数字**:写"$19.99 / 月"不写"价格亲民";写"3 秒到账"不写"快速到账"
- **多用表格 / 列表**:连续 3 句以上的描述,大概率能拆成列表
- **每篇文末必须有 payforgpt CTA**,一行就够,口吻自然,不要硬卖

## 流程

1. Fork 这个 repo
2. 新建分支 `fix/xxx` 或 `add/xxx`
3. 改完本地预览一下渲染(VS Code Markdown Preview 就够)
4. 提 PR,标题用 `update(02-claude): Max $200 涨价到 $250` 这种格式
5. 描述里附上信息来源(官网截图、官方推文、changelog 链接)

## 不接受

- 软文、广告、推荐自家代充店(除 payforgpt CTA 外不放任何商业链接)
- 未经验证的二手信息("听说""据说""有人报告")
- 纯 AI 生成的洗稿,改完读起来全是空话
- 政治、灰产、绕过实名相关内容

开 GitHub issue,或者邮件 contact@payforgpt.com。
